Projects

promster - A Hapi & Express Prometheus exporter mid 2018

A Express middleware (or Hapi plugin) to easily export request timing metrics from an Express/Hapi appliaction to Prometheus. The library contains few but powerful abilities to configure e.g. labels or to perform normalization on measured values.

react-memoise - Declaratively memoise a computation in a React component start 2018

Expensive computations should be memoised. In React projects memoisation is often performed imparatively whereas it could and likely should also be declarative within the tree of components. React-memoise enables this through an render-prop based API.

flopflip - Real time feature toggling: through LaunchDarkly in React and/or Redux via HoCs mid 2017

A library with multiple packages in a monorepo (maintained with Lerna) offering Higher Order Components (HoCs) and other utilities to enable easy feature toggling of React components (supporting single- and multivariate flags). A version for Redux (via a store enhancer) integrates nicely with the Redux DevTools. Another standalone React version using a broadcasting mechanism allows integrating the library within any React application.

Colloq - A playground: four NodeJs services with GraphQL in front start 2017

A pet project with the domain model of a conference and its attendees, speakers and organisers. Build with a NodeJs service for each entity and with a GraphQL (Apollo Stack) server in front. Technologies included in this project is: GraphQL, HapiJs, Ava (testing), XO (linting), nodemon, Docker and PM2.

tamesy - Tames a set of wild concurrent promises by a concurrency limit end 2016

Tamesy exposes a map function to map over a set of Promises or a iterator factory with a given concurrency. It returns a Promise which resolves to an Array whenever all tasks have ben run. The order of items is maintained - same as in the list of factory functions passed. It is built using webpack as a UMD module and runs on Node.js or in the browser as CommonJS, AMD, ESM or as a global.

dactylographsy - A set of tools generating manifests with fingerprinted assets to be injected and cached mid 2015

Every unit of an application can create a custom manifest listing all assets it consists of. These assets will be injected into the page while their contents will be cached when they are once loaded. After the cache (localstorage) is filled it serves all files on any subsequent page load to speed up the initial load time. The caches might be invalidated at runtime by making a comparison between old and eventually new manifest(s).

DOMPurify - A DOM-only, super-fast, uber-tolerant XSS sanitizer for HTML, MathML and SVG mid 2015

Contributor for DOMPurify which is written in JavaScript and works in all modern browsers (Safari, Opera (15+), Internet Explorer (10+), Spartan, Firefox and Chrome - as well as almost anything else using Blink or WebKit). It doesn't break on IE6 or other legacy browsers. It simply does nothing there. Helping to ensure code quality for setting up and maintaining continous integration via TravisCI and BrowserStack in 8 different browsers.

grunt-voguesy - A grunt task to analyse the up-to-dateness of your dependencies early 2015

A grunt-task (build-step) allowing to specify up-to-dateness requirements for npm and bower dependencies in regards to Semantic Versioning (major.minor.patch) with thresholds.

infinite-scrollsy - AngularJS module for e.g. infinite scrolling late 2014

An performant module for AngularJS allowing to attach event handlers to elements being triggered whenever a scroll threshold is passed. In most of the case it will be used for infinite scrolling.

grunt-weightwatchsy - Grunt task for gathering the file sizes of different asset types over time. mid 2014

A grunt-task (build step) aggregating and grouping asset sizes of various types over time for graphing while also allowing to specify size budgets breaking a build.
